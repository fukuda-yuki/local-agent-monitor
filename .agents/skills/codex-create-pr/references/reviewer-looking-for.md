# レビュー観点（要約）

## 正確性

- 仕様や期待挙動に一致し、バグや抜け漏れがない。
- エッジケースや例外パスが考慮されている。

## 設計

- 置き場所と責務が妥当で、既存システムと整合する。
- 影響範囲と依存関係が妥当。

## 複雑性

- 必要以上に複雑でない。
- 将来の仮定を先回りし過ぎていない。

## テスト

- 変更を検証するテストがある。
- 壊れたら落ちるテストになっている。

## 命名/コメント

- 命名だけで役割が伝わる。
- コメントは「なぜ」を説明している。

## スタイル

- 周辺コードとの一貫性がある。
- 大規模な整形を機能変更に混ぜていない。

## ドキュメント

- 影響があれば README/ドキュメントが更新されている。
