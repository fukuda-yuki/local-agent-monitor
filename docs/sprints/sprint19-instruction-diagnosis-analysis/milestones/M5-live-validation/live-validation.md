trace, not as a full conversation transcript in a single trace.

## Observations (recorded for Phase 2 consideration)

1. Category=evidence coupling is the weakest contract element: 2 of 9
   findings (8/3, 8/4) cite real trace evidence but stretch the category
   definition (single-goal exploration overhead labeled `task-size-split`;
   self-reported agent uncertainty labeled `ambiguity` without a user
   rephrase turn). The three formal gate criteria still hold; a Phase 2
   prompt or deterministic pre-extraction should tighten the coupling.
2. Analysis scope is a single trace id while Copilot CLI splits a
   conversation across traces (one per invocation, linked by conversation
   id). Diagnosis of first-instruction defects therefore often lands on the
   follow-up-turn trace (A2, B2, C2 pattern). Conversation-scoped analysis is
   a natural Phase 2 candidate.
3. The runner concatenates every assistant delta, so pre-report tool
   narration leaks into `result_markdown` (runs 7, 8, 10), and run 10's tail
   paragraph was truncated at stream end. Cosmetic, not gate-relevant.
4. Live provider variance is real: `deepseek-v4-flash` (validated 2026-07-04)
   was returning provider-side 500s during this validation; `minimax-m2.7`
   rejects the SDK tool-call wire shape. `glm-5.2` on the OpenCode Go
   endpoint is the currently validated analysis model on this machine.
