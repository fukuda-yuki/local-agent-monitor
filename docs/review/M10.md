## メインエージェントの妥当性判断

- 仕様準拠性: 妥当。scan は git 履歴全体、Issue は毎回新規、Issue body は secret 値 / match 文字列 / 前後文脈 / report 全文を含めない。
- 権限: 妥当。workflow permissions は `contents: read` と `issues: write` に限定している。
- failure handling: 妥当。gitleaks finding は `--exit-code 2` で scan error と分離し、finding 時は Issue 作成後に成功扱い、scan error や Issue 作成失敗は失敗扱いになる。
- 再現性: 妥当。gitleaks CLI version と checksum を固定している。
- 情報露出リスク: 残る。commit link と Fingerprint はユーザー指定により Issue に載せるが、公開 repository では該当 commit の secret 位置特定を容易にする。

## 対応方針

- 追加修正不要。
- workflow の実動作は default branch 反映後に `workflow_dispatch` で確認する。
