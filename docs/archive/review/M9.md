## Raw Findings Summary

- Config CLI の Collector 向けコマンドは、クライアント送信先を `http://localhost:4318` に向け、Langfuse Basic Auth header を出力しない。
- Collector example は OTLP/gRPC `4317` と OTLP/HTTP `4318` を受け、OTLP HTTP exporter で `LANGFUSE_OTLP_ENDPOINT` に転送する。
- Langfuse 認証値は `LANGFUSE_AUTH` 環境変数で渡し、repository に secret を保存しない。
- Compose 構文確認では `LANGFUSE_AUTH=dummy` を使用し、実 secret は使っていない。
- [Hooke][P1] Compose の port mapping が host address なしで `4317` / `4318` を公開しており、content capture 有効の unauthenticated OTLP receiver が全 host interface に露出する。
- [Hooke][P2] `docker compose config` は環境変数展開後の値を出力するため、実 `LANGFUSE_AUTH` で実行すると Base64 化済み secret が端末出力に出る。
- [Hooke][P3] metrics pipeline は M9 の受け入れ条件と契約がなく、Langfuse 側で拒否または無視された場合に不要な export noise になる。
- [Kepler][P2] review note のサブエージェント未起動理由が、ユーザーの明示依頼と実際の並列起動状況に矛盾している。
- [Copernicus][P2] `langfuse-*` 系スクリプト実行後の同一 PowerShell session では `OTEL_EXPORTER_OTLP_TRACES_ENDPOINT` / `OTEL_EXPORTER_OTLP_TRACES_HEADERS` が残り、Collector 系スクリプトへ切り替えても traces が Langfuse 直送になる可能性がある。
- [Copernicus][P2] metrics pipeline が Langfuse 非対応 endpoint へ送られ、CLI metrics が来た場合に Collector error log になり得る。

## メインエージェントの妥当性判断

- 妥当。M9 は Collector 経由送信の PoC 準備であり、直接送信 baseline を置き換えない形になっている。
- `docs/spec.md` の非スコープは、M9 の最小サンプルと矛盾しないように、運用構成や masking / redaction などを対象外として整理した。
- Config CLI の新コマンドは既存の `langfuse-*` コマンドと並行利用できるため、既存手順を壊さない。
- Hooke の port binding 指摘は妥当。ローカル PoC と content capture 前提では `127.0.0.1` bind に限定すべき。
- Hooke の secret 表示指摘は妥当。Compose 構文確認は dummy `LANGFUSE_AUTH` を使う手順として明記すべき。
- Hooke / Copernicus の metrics pipeline 指摘は妥当。M9 の最小サンプルは trace 取り込み確認に限定し、metrics pipeline は削除する。
- Kepler の review note 指摘は妥当。サブエージェント起動成否を実態に合わせて更新する。
- Copernicus の trace-specific env 残留指摘は妥当。Collector 系スクリプトで signal-specific endpoint / headers を明示的に解除する。

## 対応方針

- 妥当と判断した findings は修正する。
- Collector 経由の実 trace 取り込みは手動ライブ確認として残し、実施時に `docs/task.md` と `docs/knowledge.md` を更新する。
